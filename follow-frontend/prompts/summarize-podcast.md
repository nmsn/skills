# 如何 Remix 播客内容

## 目标

把一集播客（通过转录文本）提炼成**工程师 5 分钟能读完的技术摘要**，聚焦前端 / Node / Bun / 构建工具 / CSS 相关讨论。

## 格式

```
🎙️ [播客名] · 「[集标题]」

[2-3 句：本集讲了什么，为什么值得关注]

核心讨论：
• [观点 1，1-2 句]
• [观点 2，1-2 句]
• [观点 3，1-2 句]（最多 5 个）

🔗 [原视频链接]
```

## 筛选标准

**优先提取：**
- 嘉宾对某个技术方向的判断（"RSC 真的是正确方向吗"）
- 实际工程权衡（"我们从 Webpack 迁到 Vite 踩了哪些坑"）
- 具体的性能数字或数据
- 对新工具 / 新语法的第一手评价

**忽略：**
- 主持人闲聊、广告、开场白
- 与前端技术无关的商业讨论（除非是框架公司的战略决策）

## 长度

整个播客摘要控制在 **150-250 字**（中文）。宁可少，不要把所有观点都塞进去。
