# Project History Analyzer Skill

## 1. Concept & Vision

A skill that tells the story of an open-source project's evolution through its commit history, tags, releases, and milestones. The output reads like a narrative — tracing the project's journey from birth to present day, highlighting pivotal moments, release rhythms, and community growth. The goal is to help users understand a project's trajectory without reading hundreds of commit messages.

**Trigger:**
- Conversation: 「分析一下 facebook/react 的发展历程」
- Slash command: `/project-history facebook/react`

**Output:** A Markdown report written to `{owner}-{repo}-history.md` in the current directory, with a narrative summary displayed in the chat.

---

## 2. Data Sources (gh CLI)

All data is fetched via `gh api`. No local git clone required.

| Data | gh command |
|------|------------|
| Basic repo info | `gh repo view {owner}/{repo}` |
| Commits (paginated) | `gh api repos/{owner}/{repo}/commits` |
| Tags | `gh api repos/{owner}/{repo}/tags` |
| Releases | `gh api repos/{owner}/{repo}/releases` |
| Milestones | `gh api repos/{owner}/{repo}/milestones` |
| Contributors | `gh api repos/{owner}/{repo}/contributors` |
| Issues (for activity) | `gh api repos/{owner}/{repo}/issues --state all` |

**Pagination:** Use `--paginate` or per_page=100 with multiple requests.

---

## 3. Output Structure (Markdown Report)

```markdown
# {Repo Name} — Project History

> {owner}/{repo} | Generated {date}

## Overview
- Created: {date}
- Total commits: {N}
- Latest version: {tag}
- Active contributors: {N}

## Version Release Rhythm
Analysis of time between major releases, identifying patterns (e.g., "major release every 6 months").

## Key Milestones
Chronological narrative of each major version/tag, describing:
- What changed
- Why it mattered
- Notable commits or decisions

## Top Contributors
| Rank | Contributor | Commits |
|------|-------------|---------|
| 1 | username | N |

## Community Activity Trends
Issue/PR open/close trends over time, highlighting periods of high/low activity.

## Summary
Brief AI-generated narrative tying the story together.
```

---

## 4. Technical Notes

- Use `--jq` flag for `gh api` to filter fields and reduce output size.
- Narrative text for each milestone is generated by summarizing commit messages between tags.
- Report filename: `{owner}-{repo}-history.md` (e.g., `facebook-react-history.md`).
- Error handling:
  - Repo not found → clear error message
  - Private repo → prompt to check authentication
  - Rate limited → show retry suggestion

---

## 5. Implementation Scope (MVP)

**In scope:**
- Parse `owner/repo` from natural language or slash command
- Fetch data via `gh` for public repositories
- Generate Markdown report with all 5 sections above
- Display narrative summary in chat after writing file

**Out of scope for MVP:**
- Local git clone fallback
- Charts/visualizations (future enhancement)
- Interactive Q&A mode
- Caching / incremental updates
